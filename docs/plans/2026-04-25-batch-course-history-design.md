# 视频合集笔记历史分组设计

## 背景

用户复制到生成入口的链接可能是单个视频，也可能是一个视频合集链接。当前合集生成出的子视频笔记会像普通单视频一样平铺在历史列表中，导致查看和删除时缺少合集维度，不便统一管理。

## 目标

- 普通单视频笔记继续在历史列表中平铺展示。
- 视频合集链接生成的笔记按合集分组展示。
- 用户可以在合集下查看各个子视频笔记。
- 用户可以删除整个合集，也可以只删除合集中的某个子视频笔记。
- 保持现有单视频生成、查看、删除流程不被破坏。

## 方案选择

采用“复用现有 batch 机制，补齐合集分组能力”的方案。

不新增独立 course 表，先把视频合集链接生成出来的一组子视频任务视为一个稳定的 batch 分组。后端围绕 batch 提供合集级元信息、子任务列表、状态统计和删除能力；前端历史列表按 batch 分组展示。

这个方案改动集中，能复用现有批量生成链路，风险低。后续如果需要合集重命名、合集导出、课程编辑等更强能力，再把 batch 演进为独立 Course 模型。

## 后端设计

### 合集分组元信息

后端在 batch 记录或 batch 查询响应中提供：

- `batch_id`：合集分组标识。
- `title`：合集标题；缺失时可显示“视频合集”或“批量任务”。
- `source_url`：用户提交的原始合集链接。
- `items`：子视频任务列表。
- `total_count`：子视频总数。
- `completed_count`：已完成数量。
- `failed_count`：失败数量。
- `running_count`：生成中数量。
- `status`：合集整体状态。

单个子视频仍沿用现有 note task 流程，合集分组只做父级聚合，不改写单个笔记生成逻辑。

### 查询接口

后端需要提供能恢复历史分组的数据：

- 查询普通任务历史。
- 查询 batch/合集历史。
- 查询指定 batch 的子任务列表和状态统计。

前端刷新页面或重新打开时，根据后端返回的 batch 信息还原“合集分组 + 子笔记”结构；没有 `batch_id` 的旧任务继续作为普通笔记展示。

### 删除接口

删除分两层：

1. 删除整个合集。
   - 删除 batch 分组记录或 batch JSON。
   - 删除该 batch 下所有子任务的 DB 记录。
   - 删除子任务状态文件、结果文件和缓存文件。
   - 返回删除结果，前端据此移除整个合集分组。

2. 删除合集中的单个子视频笔记。
   - 只删除该子任务对应的 DB 记录和文件。
   - 保留 batch 分组。
   - 重新计算合集统计信息。
   - 如果合集下没有子项，前端可显示空合集或直接移除合集；优先直接移除，减少无意义历史项。

## 前端设计

### 历史列表结构

`History/NoteHistory` 展示混合列表：

- 普通单视频笔记：沿用现有平铺展示。
- 合集分组：以父项展示，支持展开/收起。

合集父项展示：

- 合集标题。
- 来源平台或来源链接。
- 总视频数。
- 已完成、失败、生成中数量。
- 展开/收起操作。
- 删除合集操作。

展开合集后展示子视频笔记列表。每个子项展示：

- 视频标题。
- 当前生成状态。
- 更新时间。
- 查看笔记操作。
- 删除该笔记操作。

### 查看行为

- 点击合集父项默认展开或收起。
- 点击子视频笔记时，右侧 Markdown 查看器显示该子视频的笔记内容。
- 正在生成中的子项展示状态，不进入最终笔记视图。
- 普通单视频笔记的查看行为保持不变。

### 轮询行为

合集生成中时，前端轮询 batch 状态并更新父级统计和子任务状态。子任务不再作为一堆独立历史项持续刷入列表，避免历史列表被合集子视频打散。

## 兼容性

- 旧的单视频任务没有 `batch_id`，继续作为普通笔记展示。
- 旧的 batch 如果缺少合集标题，前端退化显示“视频合集”或“批量任务”。
- 单视频生成、查看、删除接口保持原有语义。
- 合集分组能力建立在 batch 之上，避免引入数据库迁移风险。

## 错误处理

- 查询 batch 历史失败时，前端保留已有本地历史，不清空列表，并提示刷新失败。
- 删除合集失败时，前端不移除合集项，提示用户重试。
- 删除子视频失败时，前端不移除子项，提示用户重试。
- 部分子任务失败时，合集父项显示失败数量；成功的子视频仍可查看。

## 测试计划

### 后端

- batch 创建后能返回 `batch_id`、子任务和合集统计。
- batch 查询能返回父级信息和子任务列表。
- 删除合集会清理 batch 记录、子任务记录、状态文件、结果文件和缓存文件。
- 删除单个子任务后，父级统计会更新。
- 单视频任务没有 `batch_id` 时仍能正常查询和删除。

### 前端

- 历史列表能同时展示普通笔记和合集分组。
- 合集可展开/收起，子视频笔记可点击查看。
- 删除合集后整个分组从历史列表移除。
- 删除子视频后仅该子项移除，父级统计更新。
- 旧单视频历史展示和查看不受影响。

### 回归

- 单视频生成到查看的完整流程保持可用。
- 合集生成过程中，历史列表不会被子任务平铺打散。
- 页面刷新后能恢复合集分组结构。
